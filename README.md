# N-Book 图书管理系统 python633

## 项目概述

N-Book 是一款现代化的图书管理系统，基于前后端分离的架构设计，提供图书管理、用户管理、借阅管理以及通知系统等多元化功能。

## 技术栈

### 后端 (Backend)

- **框架**: Flask 2.3.3
- **数据库**: MySQL + SQLAlchemy ORM
- **认证**: JWT (Flask-JWT-Extended)
- **API**: RESTful API (Flask-RESTful)
- **缓存**: Redis
- **文件处理**: Pillow
- **数据库迁移**: Flask-Migrate

### 前端 (Frontend)

- **框架**: Vue 3.3.4
- **路由**: Vue Router 4.2.4
- **状态管理**: Pinia 2.1.6
- **UI组件**: Element Plus 2.3.8
- **图表**: ECharts 5.4.3
- **HTTP客户端**: Axios 1.4.0
- **构建工具**: Vite 4.4.9

## 功能模块

- 图书管理
  - 图书信息维护
  - 图书分类管理
- 用户管理
  - 用户信息管理
  - 用户权限控制
- 借阅管理
  - 借阅记录
  - 还书提醒
- 通知系统
  - 借阅通知
  - 系统公告

## 系统角色

- 系统管理员
- 图书管理员
- 普通用户

## 运行环境

- 后端: Python 3.8+，Flask 2.3.3，MySQL 5.7+
- 前端: 现代浏览器

## 部署步骤

1. 准备开发环境
   - 安装Python及依赖
   - 安装MySQL数据库
   - 安装Redis
2. 克隆代码仓库
3. 根据文档配置前后端
4. 运行数据库迁移
5. 启动后端服务
6. 构建前端项目
7. 部署至Web服务器

## 目录结构

```
/backend              # 后端代码
/frontend             # 前端代码
  /src
    /api               # API接口定义
    /assets            # 资源文件
    /components        # 组件
    /views             # 页面
    /store             # 状态管理
```

## 核心亮点

- 基于Flask和Vue的前后端分离架构，易于维护和扩展
- 使用JWT进行用户认证，保障系统安全
- RESTful API设计，便于与其他系统集成
- 高效的缓存机制，提升系统响应速度
- 完善的文档支持，降低部署难度
